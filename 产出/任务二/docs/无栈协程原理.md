# 无栈协程原理

## 概念

无栈协程（stackless coroutine）是一种可在执行过程中暂停并在暂停处恢复的函数。它与有栈协程的关键区别在于：无栈协程**不持有独立的函数调用栈**，执行时借用当前 OS 线程的栈，暂停时释放栈空间，只将跨越暂停点的局部变量保存在堆上的状态机对象中。

Rust 的 `async`/`await` 是典型的无栈协程实现，编译器在编译期将 `async` 函数变换为一个匿名状态机（实现 `Future` trait），每个 `.await` 调用点对应状态机的一个状态转移。

## 编译器变换

### 状态机生成

以下 `async` 函数：

```rust
async fn example() {
    let a = 1;
    foo().await;
    let b = 2;
    bar().await;
    println!("{}", b);
}
```

被编译器变换为类似如下的状态机：

```rust
enum ExampleStateMachine {
    State0 { a: i32 },
    State1 { a: i32, fut_foo: FooFuture },
    State2 { b: i32 },
    State3 { b: i32, fut_bar: BarFuture },
    Done,
}

impl Future for ExampleStateMachine {
    type Output = ();
    fn poll(self: Pin<&mut Self>, cx: &mut Context) -> Poll<()> {
        loop {
            match self {
                State0 { a } => { /* 初始化 */ }
                State1 { a, fut_foo } => {
                    ready!(fut_foo.poll(cx));
                    // 继续执行
                }
                // ...
            }
        }
    }
}
```

关键点：

- 每个 `.await` 将函数拆分为独立的子状态机
- 跨越 `.await` 的变量（如上面的 `b`）被提升到状态机结构体中（堆上）
- 不跨越 `.await` 的变量（如临时变量）仅存在栈上，由编译器管理
- `poll` 方法每次调用时从上一次暂停的状态继续执行

### 自引用问题

当跨越 await 的变量之间存在自引用时（如一个局部变量持有对另一个局部变量的引用），问题变得复杂：

```rust
async fn problem() {
    let s = String::from("hello");
    let r = &s;          // r 引用 s
    some_future.await;   // 暂停点
    println!("{}", r);   // 跨越 await 使用引用
}
```

编译后的状态机中，`s` 和 `r` 都需要保存在状态机结构体中。但 `r` 指向 `s`，而状态机结构体可能被移动（如 `mem::swap`），导致 `r` 成为悬垂指针。这就是 `Pin` 存在的理由——将状态机固定在内存中，禁止移动。

## Waker 与协作式调度

`Future::poll` 接收一个 `Context` 参数，其中封装了 `Waker`。当 Future 无法立即完成时（返回 `Poll::Pending`），它需要保存 Waker 并在可继续执行时通过 `Waker::wake()` 通知执行器重新 poll。

```rust
fn poll(self: Pin<&mut Self>, cx: &mut Context) -> Poll<Self::Output> {
    if data_ready() {
        Poll::Ready(result)
    } else {
        // 注册 waker，数据就绪时通知执行器
        self.waker = Some(cx.waker().clone());
        Poll::Pending
    }
}
```

这个机制的核心特征是**协作式**：Future 主动决定何时让出执行权（返回 Pending），执行器不能强制打断。这与内核线程的抢占式调度形成对比。

## 执行器

执行器（executor）管理一组 Future，通过不断调用 `poll` 推进它们直到完成。一个极简执行器的核心逻辑：

```rust
fn run(&mut self) {
    while let Some(mut fut) = self.futures.pop_front() {
        match fut.as_mut().poll(&mut cx) {
            Poll::Pending => self.futures.push_back(fut),
            Poll::Ready(_) => {},  // 完成，丢弃
        }
    }
}
```

## 与有栈协程的对比

| 维度 | 无栈协程（Rust async） | 有栈协程（Go goroutine） |
|------|----------------------|------------------------|
| 栈 | 借用线程栈，暂停时不占用 | 独立栈（初始 2KB，动态增长） |
| 状态保存 | 编译器生成状态机 | 保存所有寄存器+栈帧 |
| 切换开销 | 函数调用级（ns） | 寄存器保存/恢复（ns~μs）|
| 内存开销 | 仅状态机变量（数十字节） | 完整栈（至少 2KB） |
| 跨暂停点借用 | 支持（Pin 保证安全） | 天然支持 |
| 语言支持 | 需要编译器 async/await 支持 | 运行时即可实现 |
