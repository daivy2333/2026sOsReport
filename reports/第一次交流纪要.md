# 第一次交流纪要

## 一、训练营目标设定与考核标准

### 分层级达成目标

**基础目标（必达）**：完成前两个阶段的练习，即阅读并分析最新代码，撰写清晰的分析文档，并能完整执行整个流程。

**进阶目标（推荐）**：在理解代码的基础上，对现有模块进行力所能及的改进。

**优秀目标（挑战）**：在项目中做出具有独到见解的重大改进，或实现创新性功能。

### 核心任务与产出

1. **周报制度**：在 GitHub 建立个人仓库，每周记录学习进展与代码实践，作为过程考核依据。
2. **最终报告**：训练营结束时提交总结报告，重点展示对异步机制的理解及在项目中的具体实践成果。

## 二、个人目标定制

挑战自己，争取实习机会

## 三、技术答疑要点

### 异步与抢占的兼容性

协程因复用堆栈导致无法直接抢占，但可通过混合模型（线程+协程）解决：
- 主动让权时复用堆栈
- 被抢占时保留堆栈现场

### 硬实时调度实现

目前 rCore 主要使用 CFS 调度器，硬实时调度算法（如 C 语言实现）移植到 Rust 并与异步结合是可行的研究方向。

### 实习机会

训练营合格学员有机会进入泉城实验室进行为期 3 个月的实习，参与操作系统内核开发，可获得实习证明与补助。
